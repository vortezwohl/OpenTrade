已确认的真实问题

1. 顶层 search 命令当前不可用

真实执行 efinance search AAPL 时，命令会在内部调用 get_command_spec("utils", "search_quote")，但注册中心的函数名使用的是 search-quote 对应的 Python 函数 search_quote 之外的另一套命令名映射， 最终直接触发 KeyError('未知命令: utils.search_quote')。

  • 影响范围：所有顶层 search 调用，包括 watch、输出与落盘场景。
  • 风险等级：高。因为这是顶层发现入口，用户最先接触到的命令就会失败。
  • 验证依据：测试中保留了对该异常的显式断言，且独立复现一致。

2. --count 参数在非 watch 模式下语义不一致

在统一运行时参数里，命令接受 --count，但执行层的 default_watch_count(enabled=False, count=...) 会直接返回 1，导致 用户即便传入 --count 2，最终请求对象里也不是 2，而是 1。

  • 影响范围：全部附带统一运行时参数的叶子命令。
  • 风险等级：中。不会导致崩溃，但会让参数行为与帮助文案、用户预期不一致。
  • 验证依据：全量 CLI 参数回归里唯一失败项即为该行为。
